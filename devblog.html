Log 1: Everything So Far | 5/26/26


What the game will be

Development of the game has been going on for over 6 months now with variable productivity. The name "ExtractionTruck" is definitely a placeholder as the game is shaping towards something very different than whatever I was thinking with that title. Here is the progress so far:

I started development with character controls and car driving. My original idea was Zelda-like with extraction shooter elements that involved a car. I have since realized that not only does this idea feel a little incoherent, any linearly designed 3D game is going to be nearly impossible to make as a solo developer.

Which leads us to where the game is today, which is still entirely asset-less but has a lot of the framework complete. Essentially the game is a "3rd Person Tower Defense" with the gimmick being you can, and are encouraged to, move turrets around with full physics. The trickiest things so far have been programming the inventory system and the trajectory of projectiles. Also it took a while for me to realize i dislike Godot's signals greatly and to pivot to not using them at all.

Anyway, there is not a lot of interesting stuff to see yet but I wanted to plant this devblog here as a starting point. Hopefully I will remember to update this every couple weeks or so from now on.
